# 硬件要求

URL: https://docs.monad.xyz/node-ops/hardware-requirements

> **⚠️ 警告**
> 
> 官方不支持基于云的环境。

## 要求

除带宽外，验证器（共识参与者）和全节点之间的所有要求都相同：

- **CPU**：16 核 CPU，基础时钟速度 4.5 GHz+，例如 AMD Ryzen 9950x、AMD Ryzen 7950x、AMD EPYC 4584PX 等
- **内存**：32 GB+ RAM
- **存储**：
- 2TB 专用磁盘用于 TrieDB（执行）
- 2TB 磁盘用于 MonadBFT / 操作系统
- 两者均为 PCIe Gen4x4 NVME SSD 或更好
- **带宽**：
- 300 Mbit/s（验证器）
- 100 Mbit/s（全节点）

## 为什么使用裸机？

由于系统严格的性能和时序要求，Monad 节点必须在裸机服务器上运行，而不是虚拟化或基于云的环境（例如 AWS EC2、GCP、Azure）。

裸机服务器为节点提供对硬件的直接、稳定访问，确保与网络的平稳运行和同步：

- Monad 的共识协议强制执行严格的时间窗口——区块在亚秒级间隔内被提议和投票，网络假设节点可以在此预算内验证和执行区块。在这种情况下，基于云的环境可能会引入延迟和不可预测性，这可能导致节点错过截止日期、在区块处理中落后或在高吞吐量期间变得不稳定。
- 即使资源在纸面上看起来足够，虚拟化也会在节点和物理硬件之间添加额外的软件层。该层引入了上下文切换开销，并限制了对 SSD 和网络接口的直接 I/O 访问。这些影响对于普通计算任务来说微不足道，但当需要持续的高吞吐量和低延迟操作时（如 Monad 的共识和执行循环）就变得重要。

总之，裸机服务器提供可预测性和确定性，这对于维护整个网络的同步和吞吐量至关重要。云托管的虚拟机可能在轻负载下工作，但无法保证 Monad 共识在规模化时所需的一致实时性能。
