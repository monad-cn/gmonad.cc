# Monad 上的 EIP-7702

## 概要

[EIP-7702](https://eips.ethereum.org/EIPS/eip-7702) 在 Monad 上得到支持，工作流程与以太坊相同：用户签署消息授权委托给特定账户，他们或其他人可以使用交易类型 `0x04` 提交。

当账户在 EIP-7702 下被委托时，它们在 Monad 的[保留余额](https://docs.monad.xyz/developer-essentials/reserve-balance)规则下的处理会略有变化。具体来说，对于第一个正在进行的"清空"交易的保留余额策略有一个例外，但这不适用于 EIP-7702 委托账户。

除此之外，EIP-7702 委托账户在由 EOA 发送的普通交易和将 EOA 视为智能合约的交易方面都表现正常。

本页面的其余部分作为 EIP-7702 的入门指南，不是 Monad 特定的。

## 介绍

EIP-7702 允许外部拥有账户（EOAs）为自己添加代码，赋予自己以前只有智能合约账户才有的新功能，如交易批处理、gas 赞助和替代认证。

为此，EOA 签署授权，指定特定地址作为其代码源。EIP-7702 引入了一种新的交易类型（类型 `0x04`）来提交此授权。授权可以由 EOA 自己提交，也可以由其他任何人提交。

## 背景

EIP-7702 的创建很大程度上是为了应对 EIP-4337 的局限性，EIP-4337 是以太坊迈向账户抽象的第一个重要步骤。

虽然 EIP-4337 引入了具有灵活验证逻辑的智能合约钱包，但它依赖于替代的"用户操作"内存池和独立的打包器基础设施，使得钱包、dapp 和基础设施提供商采用起来更加复杂。这种分离意味着仍占大多数的 EOA 无法直接受益于账户抽象功能，除非完全迁移到新的智能账户，这在生态系统中造成了摩擦和碎片化。

EIP-7702 通过使 EOA 能够在现有交易模型内作为智能账户运行来解决这些不足，无需并行内存池或强制用户切换地址。

本质上，EIP-7702 使今天的 EOA 能够获得智能钱包般的功能，如多重签名、会话密钥和 gas 赞助，**而无需放弃其当前账户**，在旧的 EOA 模型和完全账户抽象的未来之间架起了桥梁。

## 使用 viem 发送 EIP-7702 交易

```ts
import { createWalletClient, http, parseEther } from 'viem'
import { monadTestnet } from 'viem/chains'
import { privateKeyToAccount } from 'viem/accounts'


const account = privateKeyToAccount('0x...')


const walletClient = createWalletClient({
  account,
  chain: monadTestnet,
  transport: http(),
})


const authorization = await walletClient.signAuthorization({
  account,
  contractAddress: '0xFBA3912Ca04dd458c843e2EE08967fC04f3579c2'
})


const hash = await walletClient.sendTransaction({
  authorizationList: [authorization],
  data: '0xdeadbeef',
  to: walletClient.account.address,
})
```
