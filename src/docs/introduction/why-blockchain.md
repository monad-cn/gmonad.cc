# 为什么选择区块链？

区块链是一组多样化参与者之间关于以下两件事的去中心化共识：

1. 交易的官方**排序**（账本）
2. 世界的官方**状态**，包括账户余额和各种程序的状态。

在以太坊等现代区块链中，交易包括余额转账、新程序的创建以及对现有程序的函数调用。迄今为止所有交易的累计结果产生了当前状态，这就是为什么*对（1）的共识意味着对（2）的共识*。

区块链系统有一套协议规则，也称为共识机制，描述了一组当前同步的分布式节点如何相互通信，以商定将哪些额外交易添加到账本中。（[MonadBFT](https://docs.monad.xyz/monad-arch/consensus/monad-bft) 是一个共识机制的示例。）

归纳法保持节点同步：它们从相同的状态开始，应用相同的交易，因此在应用一组新交易后，仍然保持一致的状态。

共享的全局状态支持去中心化应用（dApps）的开发——这些应用“存在于区块链上”，即存在于区块链系统的每个节点上。去中心化应用是一段代码（以及持久的、应用特定的状态），任何用户都可以通过提交指向该应用函数的交易来调用它。区块链中的每个节点负责正确执行被调用的字节码；重复执行确保每个节点的诚实性。

## 示例应用

去中心化应用可以实现我们通常期望以中心化方式实现的功能。例如，一个非常简单的去中心化应用示例是*虚拟银行*（在加密领域通常称为借贷协议）。

在现实世界中，银行是一家接受存款并以较高利率放贷的企业。银行赚取高利率与低利率之间的利差；借款人获得贷款以进行经济上富有成效的活动；存款人则从存款中获得利息。每个人都受益！

虚拟银行只是一个具有四个主要方法的应用程序：`deposit`（存款）、`withdraw`（取款）、`borrow`（借款）和`repay`（还款）。这些方法的逻辑主要是簿记工作，以确保存款和贷款被正确跟踪：

在以太坊或 Monad 中，有人可以为这个虚拟银行编写代码并上传；然后，任何人都可以利用它进行借贷，可能比在自己的国家获取银行服务更加容易。

这个简单的例子展示了去中心化应用的强大之处。以下是其他一些值得强调的优点：

- **开放的 API / 可组合性**：去中心化应用可以被其他去中心化应用原子化调用，允许开发者通过组合现有组件构建更复杂的功能。
- **透明性**：应用逻辑完全通过代码表达，任何人都可以审查代码是否存在副作用。状态是透明且可审计的；在去中心化金融（DeFi）中，储备证明是默认设置。
- **抗审查与可信中立**：任何人都可以向无许可网络提交交易或上传应用程序。
- **全球覆盖**：任何能够访问互联网的人都可以使用关键金融服务，包括未被银行服务覆盖或服务不足的用户。
