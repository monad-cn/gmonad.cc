# 设置您的 Monad 节点


警告
无需许可的全节点入门正在 Monad 测试网上逐步推出。本页面引用了有关如何运行全节点的文档，该文档仍在发布过程中。

此 SDK 文档提前发布，以便审计员和开发人员可以了解系统的预期工作方式。如果您想尝试 SDK，请稍后查看或在 [Monad Developer Discord](https://discord.gg/monaddev) 上联系 `ken_category_labs`。

执行事件 SDK 依赖于共享内存通信系统，其中节点的 EVM 执行守护程序充当发布者。因此，要使用它，您需要 (i) 运行自己的 Monad 节点，(ii) 在与节点相同的主机上运行数据处理应用程序，以便它可以从共享内存读取数据。

首先，按照 ***全节点设置指南***（链接即将推出）来运行您的 Monad 节点。

由于执行事件发布是一个可选功能，您必须执行一些额外的设置才能在节点上启用它。这在 ***执行事件和 WebSockets 设置指南***（链接即将推出）中有介绍。

请注意，设置指南建议您安装 `libhugetlbfs` 开发包和运行时库（在 Ubuntu 中打包为 `libhugetlbfs-dev` 和 `libhugetlbfs0`）。此要求可以通过构建配置选项关闭，但默认构建设置期望此库可用。
